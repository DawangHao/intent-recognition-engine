# 项目测试方案

## 1. 测试目标

本测试方案旨在确保项目的各个组件能够正常运行，特别是：

1. **单元测试**：测试各个模块的功能是否正常
2. **集成测试**：测试灵犀技能、意图引擎和后台接口的交互是否正确
3. **端到端测试**：测试整个流程的完整性

## 2. 测试范围

### 2.1 单元测试

- 意图引擎各模块：安全围栏、上下文管理、实体提取、意图识别、任务分发
- 后端API接口：存款查询、贷款查询、访前一页纸下载

### 2.2 集成测试

- 灵犀技能清单与意图引擎的匹配
- 意图引擎返回结果与灵犀前端的处理
- 灵犀前端与后台API的调用

### 2.3 端到端测试

- 完整的用户交互流程：用户输入 → 意图引擎处理 → 灵犀前端响应 → 后台API调用 → 结果展示

## 3. 测试策略

### 3.1 测试工具

- **单元测试**：使用 pytest 框架
- **集成测试**：使用 pytest + requests 库
- **端到端测试**：使用 pytest + requests 库 + 模拟浏览器行为

### 3.2 测试环境

- 本地开发环境
- 确保后端服务运行在 http://localhost:8000

### 3.3 测试数据

- 企业名称：比亚迪、通用电气、申瑞生物等
- 用户输入：各种查询语句，包括正常查询和边界情况

## 6. 测试脚本

### 6.1 测试运行脚本

创建 `run_tests.sh` 脚本，用于运行所有测试：

```bash
#!/bin/bash

# 运行所有测试
echo "Running all tests..."
pytest -v

# 运行特定类型的测试
echo "\nRunning unit tests..."
pytest tests/unit/ -v

echo "\nRunning integration tests..."
pytest tests/integration/ -v

echo "\nRunning end-to-end tests..."
pytest tests/e2e/ -v
```

## 7. 测试注意事项

1. **不影响程序主体运行**：测试应在独立的环境中运行，不影响生产环境的运行
2. **可手工运行**：测试脚本应易于手工运行，方便开发人员在修改代码后检查
3. **覆盖关键逻辑**：测试应覆盖所有关键逻辑，确保代码的可靠性
4. **边界情况**：测试应包括各种边界情况，确保系统的鲁棒性
5. **性能测试**：对于关键接口，应进行性能测试，确保响应时间在可接受范围内

## 8. 测试结果报告

测试完成后，应生成测试结果报告，包括：

1. 测试执行情况
2. 测试通过率
3. 失败的测试用例及原因
4. 修复建议

## 9. 结论

通过本测试方案的实施，可以确保项目的各个组件能够正常运行，特别是灵犀的技能与意图引擎、后台接口之间的交互是否正确。同时，测试方案也为开发人员提供了一种方便的方式来检查代码的关键逻辑是否被篡改。
