# 新技能开发指南

## 1. 概述

本文档详细介绍了为灵犀智能体开发新技能的完整流程，包括后端API开发、前端界面更新、技能文件创建等步骤，确保开发者能够按照标准流程快速开发和部署新技能。

## 2. 开发流程

### 2.1 步骤1：确定技能需求

**操作**：
- 明确技能的功能需求
- 确定输入参数和返回值
- 设计技能的呈现形式（卡片、表格、文件等）

**示例**：
- 技能名称：查询企业结算分
- 输入参数：企业名称
- 返回值：结算分、我行结算金额(元)、预测所有银行结算金额(元)
- 呈现形式：小卡片

### 2.2 步骤2：开发后端API接口

**操作**：
1. **创建API端点**：在 `backend/api/data_query.py` 中添加新的API端点
2. **实现业务逻辑**：编写数据查询和处理逻辑
3. **测试API接口**：确保接口能够正常返回数据

**示例代码**：
```python
@router.post("/api/settlement/query")
async def query_settlement(data: dict):
    """企业结算分查询接口"""
    company_name = data.get('company_name', '') or data.get('enterprise_name', '')
    if not company_name:
        return {"code": 400, "message": "企业名称不能为空"}
    
    # 从CSV文件中查询数据
    try:
        with open(CSV_FILE_PATH, 'r', encoding='utf-8') as f:
            reader = csv.DictReader(f)
            for row in reader:
                if row['企业名称'] == company_name:
                    # 使用资质分作为结算分
                    settlement_score = float(row['资质分']) if row['资质分'] else 0.0
                    return {
                        "code": 200,
                        "data": {
                            "company_name": company_name,
                            "settlement_info": {
                                "结算分": settlement_score,
                                "我行结算金额(元)": float(row['我行结算金额(元)']),
                                "预测所有银行结算金额(元)": float(row['预测所有银行结算金额(元)'])
                            }
                        }
                    }
        # 未找到企业
        return {"code": 404, "message": f"未找到企业: {company_name}"}
    except Exception as e:
        return {"code": 500, "message": f"查询失败: {str(e)}"}
```

4. **实现API调用**：
```javascript
async function querySettlement(companyName) {
  console.log('开始查询结算分，企业名称:', companyName);
  currentAbortController = new AbortController();
  
  try {
    updateLoading('步骤1/5: 准备请求数据...');
    await new Promise(r => setTimeout(r, 200));
    
    const controller = currentAbortController;
    const timeoutId = setTimeout(() => controller.abort(), 15000);
    
    updateLoading('步骤2/5: 发送请求到服务器...');
    const response = await fetch('http://localhost:8000/api/settlement/query', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ company_name: companyName }),
      signal: controller.signal
    });
    
    clearTimeout(timeoutId);
    console.log('结算分查询响应状态:', response.status);
    updateLoading('步骤3/5: 接收服务器响应...');
    await new Promise(r => setTimeout(r, 200));
    
    const result = await response.json();
    console.log('结算分查询结果:', result);
    updateLoading('步骤4/5: 解析响应数据...');
    await new Promise(r => setTimeout(r, 200));
    
    updateLoading('步骤5/5: 渲染结果...');
    await new Promise(r => setTimeout(r, 200));
    
    removeLoading();
    stopButton.disabled = true;
    
    if (result.code === 200) {
      const cardHtml = renderSimpleSettlementCard(result.data);
      addCardToChat(cardHtml);
    } else {
      addMessage(result.message || '查询失败');
    }
  } catch (error) {
    console.error('结算分查询错误:', error);
    if (error.name !== 'AbortError') {
      removeLoading();
      stopButton.disabled = true;
      addMessage(`查询失败：${error.message}`);
    }
    throw error;
  }
}
```

## 3. 注意事项

1. **技能ID命名规范**：使用 `SKILL_XXX_V1` 格式，确保唯一性
2. **参数一致性**：确保技能文件中的参数与前端和后端的参数名称一致
3. **API端点一致性**：确保技能文件、前端和API接口文档中的API端点一致
4. **呈现形式一致性**：确保技能文件和前端中的UI组件名称一致
5. **测试覆盖**：确保测试覆盖API接口、接口调试和意图识别三种调用方式
6. **文档更新**：确保所有相关文档都得到更新

## 4. 常见问题及解决方案

1. **API接口返回404**：检查企业名称是否正确，确保企业存在于数据文件中
2. **前端不显示卡片**：检查前端代码中是否添加了对应的渲染逻辑
3. **意图引擎无法识别技能**：检查技能文件是否正确创建，以及app_intent.json是否更新
4. **参数传递错误**：检查前端和后端的参数名称是否一致
5. **端口占用**：使用 `lsof -i :8000 | grep LISTEN | awk '{print $2}' | xargs kill -9` 停止占用端口的进程

## 5. 总结

开发新技能需要按照标准流程进行，确保后端API、前端界面、技能文件和文档的一致性。通过本指南的步骤，开发者可以快速、规范地为灵犀智能体添加新技能，提高开发效率和代码质量。
