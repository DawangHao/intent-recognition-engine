# 测试计划

## 项目测试案例管理

### 1. 测试分类

#### 1.1 单元测试
- **测试目标**：验证各个模块的独立功能是否正常
- **测试范围**：单个函数、类或模块
- **测试工具**：pytest

#### 1.2 集成测试
- **测试目标**：验证模块之间的交互是否正常
- **测试范围**：多个模块的组合
- **测试工具**：pytest + requests

#### 1.3 端到端测试
- **测试目标**：验证整个系统的流程是否正常
- **测试范围**：完整的用户场景
- **测试工具**：pytest + requests

### 3. 测试执行计划

#### 3.1 测试环境
- **后端服务**：http://localhost:8000
- **前端页面**：http://localhost:8000/lingxi
- **测试工具**：pytest, requests, curl

#### 3.2 测试运行方式
1. **单元测试**：`pytest tests/unit/ -v`
2. **集成测试**：`pytest tests/integration/ -v`
3. **端到端测试**：`pytest tests/e2e/ -v`
4. **全部测试**：`pytest tests/ -v`

#### 3.3 测试结果记录
- 测试结果将通过pytest输出到控制台
- 详细的测试日志将保存在 `tests/logs/` 目录

### 4. 测试维护

#### 4.1 测试案例更新
- 当系统功能变更时，需要更新相应的测试案例
- 新增功能时，需要添加新的测试案例
- 测试案例的状态应及时更新

#### 4.2 测试数据管理
- 测试数据应与实际业务数据保持一致
- 当业务数据变更时，需要更新测试数据

#### 4.3 测试覆盖范围
- 应确保测试覆盖系统的主要功能
- 应定期检查测试覆盖情况，补充缺失的测试案例

### 5. 测试报告

#### 5.1 测试报告生成
- 每次测试运行后，自动生成测试报告
- 测试报告包含测试结果、通过率、失败原因等信息

#### 5.2 测试报告存储
- 测试报告存储在 `tests/reports/` 目录
- 按日期和测试类型分类存储
